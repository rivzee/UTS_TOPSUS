01. Problem Discovery

Root Cause Analysis (5 Whys Method) Slide 3 / 11

WHY 1 Mengapa lulusan IT kesulitan menembus seleksi kerja web developer? ▼
Gagal pada tahap penyaringan portofolio kode riil dan tes live coding teknis industri.
WHY 2 Mengapa portofolio kode dan tes live coding mereka gagal lolos? ▼
Proyek web yang mereka buat tidak menerapkan clean code, arsitektur yang aman (security), dan performa optimal.
WHY 3 Mengapa mahasiswa tidak tahu cara menulis kode berstandar industri? ▼
Proses belajar di kampus fokus pada teori kelulusan tugas akademik dasar, bukan kualitas arsitektur skala industri.
WHY 4 Mengapa mahasiswa tidak belajar secara mandiri secara optimal? ▼
Sumber belajar web sangat banyak dan terfragmentasi, memicu kelelahan informasi (information overload) & tidak ada mentor pembimbing.
WHY 5 Mengapa mahasiswa tidak memiliki mentor / code reviewer profesional? ▼
Menyewa mentor profesional atau pengulas kode industri sangat mahal dan tidak terjangkau bagi mahasiswa vokasi.
Akar Masalah Utama (Root Cause)
Tidak adanya sistem bimbingan terstruktur dan umpan balik (code review) baris-per-baris yang personal, terjangkau, dan adaptif untuk mengarahkan proses belajar coding mahasiswa IT menuju standar industri web developer.

1. GitHub MCP Server github-mcp-v1

Menghubungkan LLM otonom langsung ke repositori Git mahasiswa. Agen dapat mendownload kode, membaca commits, menulis saran inline review di Pull Request, serta membuka issue bug secara otonom.

2. YouTube & LMS MCP Server lms-search-mcp

Mencari secara real-time video tutorial berkualitas di YouTube & Coursera, menyaring konten-konten usang, lalu merangkai video terpilih menjadi silabus kurikulum personal harian mahasiswa.

3. Local Database MCP Server sqlite-progress-mcp

Menghubungkan LLM ke database progress lokal mahasiswa secara dua arah untuk menyimpan status kelulusan materi, skor evaluasi code review, & memperbarui grafik pohon keterampilan (Skill Tree).

05. Reflection & Critical Review

Reflection & Critical Review Slide 11 / 12

1. AI Paling Membantu

Integrasi Model Context Protocol (MCP)

Sangat membantu dalam merumuskan arsitektur yang aman dan otonom untuk menghubungkan LLM ke repositori GitHub mahasiswa secara langsung. Konsep ini memecahkan keterbatasan LMS pasif konvensional dengan menghadirkan inline code review otonom.

2. Kelemahan & Koreksi AI

Vulnerability Remote Code Execution (RCE)

AI sempat merekomendasikan penjalanan unit testing otonom milik mahasiswa langsung di kontainer Docker server utama. Ini beresiko tinggi! Kami mengoreksinya agar eksekusi testing berjalan di sandbox terisolasi di sisi client atau server terpisah.

3. Cara Verifikasi Output AI

Validasi Tiga Arah (Triangulasi)

Mencocokkan peta belajar buatan AI dengan panduan kurikulum standard global (Roadmap.sh), mencoba otonom kode refactoring usulan AI di kompiler lokal, & berdiskusi intensif dengan Dosen Pengampu mata kuliah RPL.

4. Dampak Tanpa AI Generatif

Mundur Menjadi LMS Pasif Statis

Tanpa AI, platform hanya akan menjadi portal materi static konvensional seperti Moodle. Kehilangan AI berarti tidak ada peta jalan adaptif individual, tidak ada feedback coding otonom PR, & tidak ada simulator interview interaktif.
